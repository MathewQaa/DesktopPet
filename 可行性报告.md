## 一、市场概述

### 1.1 市场规模与趋势

桌面宠物（Desktop Pet / 桌宠）是一种运行在用户桌面上的虚拟宠物/角色应用程序，能够提供陪伴感、情绪价值、轻度交互娱乐等功能。近年来，桌宠市场呈现以下趋势：

- **情绪经济崛起**：全球心理健康与情绪价值消费需求持续增长，虚拟陪伴类产品受到 Z 世代和千禧一代的广泛欢迎。
- **IP 化与个性化**：从早期的通用形象（如咪咪虾仔、电子狗）发展到基于用户自定义图片/AI 生成的个性化角色。
- **AI 技术赋能**：大语言模型（LLM）和生成式 AI 让桌宠从简单的动画交互进化为可以对话、学习用户习惯的智能伴侣。
- **跨平台需求**：用户对 macOS、Windows、Linux 全平台支持的需求越来越强。

## 三、技术可行性分析

### 3.1 技术栈选择

| 方案 | 技术栈 | 优点 | 缺点 | 推荐度 |
|------|--------|------|------|--------|
| **方案 A** | Electron + Canvas 2D | 跨栈、生态丰富、易于开发动画 | 资源占用较高 | ⭐⭐⭐⭐ |
| **方案 B** | Tauri + Canvas/WebGPU | 轻量、性能好、Rust 后端 | 学习曲线较陡 | ⭐⭐⭐⭐⭐ |
| **方案 C** | Swift (macOS) + Kotlin (Windows) | 原生性能、系统集成度高 | 多平台维护成本高 | ⭐⭐⭐ |
| **方案 D** | Godot 引擎 | 动画系统强大、导出多平台 | 包体较大、过重 | ⭐⭐⭐ |

**推荐方案：B (Tauri + Canvas 2D)**
- Tauri 基于系统 WebView，包体小（<10MB），内存占用低
- Canvas 2D 足以渲染 2D 精灵动画，GPU 加速
- Rust 后端可处理文件系统、网络请求、AI 对话等
- 前端可用任何 Web 框架（Vue/React/Svelte）

### 3.2 核心技术难点

| 难点 | 解决方案 | 难度 |
|------|---------|------|
| 透明窗口 + 置顶 | Tauri 支持透明窗口配置，WinAPI/macOS API 设置置顶 | 中 |
| 角色动画系统 | 帧动画 Spine/Lottie 或逐帧 Sprite Sheet | 中 |
| 用户图片 → 桌宠形象 | OpenAI Image API / Stable Diffusion 生图 + 抠图 | 中高 |
| AI 对话集成 | 调用 OpenRouter/本地 LLM API | 低 |
| 拖拽物理效果 | 简单重力 + 碰撞检测 | 中 |
| 跨平台构建 | Tauri CLI + GitHub Actions CI/CD | 低 |

### 3.3 AI 形象生成流程

```
用户上传图片
    ↓
图片分析（人物特征提取：发型、发色、服装、配饰、瞳色等）
    ↓
生成提示词（Prompt Engineering）
    ↓
调用图像生成 API → 可爱 Q 版/像素风角色
    ↓
自动抠图（移除背景 → 透明 PNG）
    ↓
导出 Sprite Sheet（各动作帧序列）
    ↓
导入桌面宠物应用
```

## 五、竞争优势分析

### 5.1 核心差异化

1. **AI 形象生成 → 桌宠一体化**：这是最大的卖点。目前市场上没有任何产品能做到"上传一张照片，自动生成专属桌宠"。
2. **跨平台原生体验**：Tauri 方案保证 macOS + Windows 原生体验，比 Electron 桌宠更轻量。
3. **AI 智能对话**：不是预设台词，而是真正的 AI 对话，每次互动都不同。
4. **开源 + 社区驱动**：开源项目可以吸引开发者贡献新形象、新动作、新语言。

### 5.2 SWOT 分析

| | 正面 | 负面 |
|---|------|------|
| **内部** | **优势**：AI 形象生成差异化、跨平台、开源生态 | **劣势**：新品牌认知度低、初期资源有限 |
| **外部** | **机会**：情绪经济持续增长、AI 技术成熟、桌宠市场空白 | **威胁**：大厂复刻风险、AI 生成版权争议 |

## 八、总结

桌宠市场目前存在明显的**产品断层**——传统桌宠功能简单、形象单一，现代桌宠缺乏 AI 能力。本项目结合 AI 形象生成 + AI 智能对话 + 跨平台桌面应用，具备强烈的差异化优势和明确的商业路径。

**核心路径**：先做 MVP（基础桌宠体验）→ 加入 AI 形象生成（核心卖点）→ 加入 AI 对话（增值点）→ 养成系统（留存点）。

建议在 6 周内完成 MVP 并在 GitHub 开源，通过早期社区反馈验证市场。

> 基于 2026-06-10 讨论更新

### 9.1 模式概述

| 层级 | 内容 | 收费 |
|------|------|------|
| **客户端（免费）** | 桌面宠物基础功能 + 1 个固定形象 | 免费 |
| **Web 形象商店** | AI 生成的个性化形象，按个/套餐付费 | 付费 |
| **AI 对话** | 内置基础对话（有限额） | 免费 + 订阅增值 |

### 9.2 模式优势

1. **零门槛获客**：免费下载 + 固定形象即可使用完整桌宠功能，用户可以零成本体验核心玩法
2. **付费意愿强**：用户已经习惯了桌宠的陪伴，为"个性化形象"付费是水到渠成的情绪消费
3. **边际成本极低**：AI 生成形象的边际成本趋近于近零，每个新增形象几乎纯利润
4. **Web 端集中管理**：形象资产集中在 Web 端，客户端轻量，更新形象库无需发版
5. **病毒式传播**：用户分享自己的个性形象 → 吸引新用户下载

### 9.5 形象生成商业化流程

```
① 用户在 Web 端上传自己的照片/参考图
        ↓
② AI 分析特征 → 提示词优化 → 多角度生成
   （等待时间 10-30 秒，可做成趣味加载动画）
        ↓
③ 用户预览 4 个候选形象，选择最喜欢的
        ↓
④ 支付 → 生成完整 Sprite Sheet（全套动作帧）
   - 待机动画 × 3
   - 行走动画 × 4 方向
   - 拖拽反应 × 2
   - 睡觉动画 × 1
   - 开心/伤心/生气表情 × 3
   - 特殊动作 × 5
        ↓
⑤ 形象绑定用户账户
   → 客户端内「下载已购形象」一键同步
   → API 验证 License（防止盗用）
```

### 9.7 关键风险与应对

| 风险 | 应对 |
